# Installation & Setup

Complete setup process for Maverick

<div class="grid grid-cols-2 gap-4">

<div>

## Prerequisites

<v-click>

- **Python 3.10+** installed
- **[uv](https://docs.astral.sh/uv/)** - Fast Python package manager (recommended)
- **GitHub CLI** (`gh`) authenticated
- **Claude API key** from Anthropic
- **[Speckit](https://speckit.org)** installed for task generation

</v-click>

<v-click>

## Installation

```bash
# Install uv (if needed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Clone repository
git clone https://github.com/get2knowio/maverick.git
cd maverick

# Install with uv (recommended - uses uv.lock)
uv sync

# Run maverick
uv run maverick --help
```

</v-click>

</div>

<div>

<v-click>

## Configuration

```bash
# Set API key
export ANTHROPIC_API_KEY=your-key-here

# Initialize config
maverick config init
```

</v-click>

<v-click>

**Config locations**:
- Project: `maverick.yaml` (version controlled)
- User: `~/.config/maverick/config.yaml` (local)

</v-click>

<v-click>

```yaml
# Example maverick.yaml
validation:
  timeout: 300
  parallel: true
workflows:
  max_agents: 3
notifications:
  enabled: true
```

</v-click>

</div>

# Generating tasks.md with Speckit

Use Speckit to generate properly formatted task files

<div class="grid grid-cols-2 gap-4">

<div>

## The Speckit Workflow

```bash
# 1. Write your feature spec (spec.md)
# 2. Create the implementation plan (plan.md)
/speckit.plan

# 3. Generate tasks from the plan
/speckit.tasks

# 4. Run maverick with the generated tasks
maverick fly my-feature \
  --task-file specs/my-feature/tasks.md
```

<v-click>

<div class="mt-4 p-3 bg-yellow-500/20 border border-yellow-500 rounded text-sm">
<strong>Important:</strong> Manual creation of tasks.md is not supported. The file format is specific and must be generated by Speckit.
</div>

</v-click>

</div>

<div>

<v-click>

## What Speckit Generates

**Structured Format**
- Task IDs (T001, T002, ...)
- Parallelization markers (`P:`)
- User story labels (`[US1]`, `[US2]`)
- File path references

</v-click>

<v-click>

**Phase Organization**
- Setup phase (dependencies, scaffolding)
- Foundational phase (core components)
- User Stories (feature implementation)
- Polish phase (tests, docs)

</v-click>

<v-click>

**Dependency Ordering**
- Tasks ordered by dependencies
- Parallel tasks identified automatically
- Clear execution path for Maverick

</v-click>

<v-click>

**Visit [speckit.org](https://speckit.org) for documentation**

</v-click>

</div>

# Best Practices

Practical tips for real-world usage

<div class="grid grid-cols-2 gap-4">

<div>

## Configuration Tips

<v-click>

**Validation Timeouts**
```yaml
validation:
  timeout: 300  # 5 min for small projects
  timeout: 900  # 15 min for large test suites
```

</v-click>

<v-click>

**Agent Limits**
```yaml
workflows:
  max_agents: 3  # Conservative for rate limits
  max_agents: 5  # If you have higher limits
```

</v-click>

<v-click>

**Team Consistency**
- Use project-level `maverick.yaml`
- Version control configuration
- Document team conventions in CLAUDE.md

</v-click>

</div>

<div>

<v-click>

## Workflow Tips

**Development Iteration**
```bash
# Preview before executing
maverick fly --dry-run

# Skip PR creation during dev
maverick fly --skip-pr

# Monitor progress
maverick status
```

</v-click>

<v-click>

## Common Pitfalls

**Branch Sync Issues**
- Always sync with main before starting
- Workflow auto-syncs by default
- Prevents merge conflicts

</v-click>

<v-click>

**Large Features**
- Split large features into smaller specs
- Let Speckit generate focused task files
- Use multiple workflow runs

</v-click>

<v-click>

**Missing Conventions**
- Create CLAUDE.md with project patterns
- Agents use it for consistency
- Update after major changes

</v-click>
