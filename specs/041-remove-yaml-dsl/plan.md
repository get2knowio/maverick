# Implementation Plan: Remove Dead YAML DSL Infrastructure

**Branch**: `041-remove-yaml-dsl` | **Date**: 2026-03-03 | **Spec**: `specs/041-remove-yaml-dsl/spec.md`
**Input**: Feature specification from `/specs/041-remove-yaml-dsl/spec.md`

## Summary

Remove ~20,000 lines of dead YAML DSL infrastructure from `maverick.dsl`, extracting the ~3,700 lines of actively-used modules (events, executor, results, types, checkpoint, registry) to proper top-level package locations, then bulk-deleting everything that remains. Also delete ~33,400 lines of dead tests (~28,000 unit + ~5,400 integration), ~6 dead YAML workflow/fragment files, dead CLI functions, dead `library/builtins.py` YAML-loading methods, the `lark` dependency, and update all imports across the codebase.

## Technical Context

**Language/Version**: Python 3.10+ (with `from __future__ import annotations`)
**Primary Dependencies**: Claude Agent SDK, Click, Rich, Pydantic, PyYAML, GitPython, tenacity, structlog
**Storage**: JSON checkpoint files under `~/.maverick/checkpoints/`
**Testing**: pytest + pytest-asyncio, parallel via xdist (`-n auto`)
**Target Platform**: macOS / Linux CLI
**Project Type**: Single Python package (`src/maverick/`)
**Constraints**: Zero behavior changes to active CLI commands; all existing Python workflow tests must pass

## Execution Phases

### Phase 1: Extract Live Modules (P1 — User Story 1)

**Goal**: Move all actively-used code out of `maverick.dsl` to new top-level locations. All imports updated. Full test suite passes.

**Sequence** (dependency-ordered):

1. **Create `maverick.types`** — No internal DSL dependencies. Strip dead type aliases (`ContextBuilder`, `Predicate`, `RollbackAction`) that depend on dead `WorkflowContext`.
2. **Create `maverick.constants`** — Extract active constants from `dsl/config.py` as module-level constants.
3. **Create `maverick.events`** — Depends on types. Update internal import from `dsl.types` → `maverick.types`.
4. **Create `maverick.results`** — Depends on types. Update internal imports.
5. **Create `maverick.executor/`** — Depends on types, events, results. Move all 5 files. Update all internal imports.
6. **Create `maverick.checkpoint/`** — Depends on constants. Move all files. Update `DEFAULTS.CHECKPOINT_DIR` → `constants.CHECKPOINT_DIR`.
7. **Create `maverick.registry/`** — Depends on executor (protocol types). Move live files only. Remove `WorkflowRegistry` and `ContextBuilderRegistry` from `ComponentRegistry`.
8. **Merge live errors into `maverick.exceptions`** — Add `WorkflowStepError` (renamed from `DSLWorkflowError`), `CheckpointNotFoundError`, `InputMismatchError`, `ReferenceResolutionError`, `DuplicateComponentError` to `maverick.exceptions.workflow`.
9. **Update all consumer imports** — Every file in the Import Update Map above.
10. **Run full test suite** — `make check` must pass.

### Phase 2: Delete Dead Code (P2 — User Stories 2 & 3)

**Goal**: Remove all dead YAML DSL code, dead CLI functions, dead YAML files, and the `lark` dependency.

**Sequence**:

1. **Delete dead CLI functions** — Remove `execute_workflow_run()`, `format_workflow_not_found_error()`, `execute_dsl_workflow()`, `get_discovery_result()`, and all dead imports from `cli/workflow_executor.py`, `cli/helpers.py`, `cli/common.py`.
2. **Clean up `library/builtins.py`** — Remove YAML-loading methods (`get_workflow`, `get_fragment` on the concrete class that calls `parse_workflow`). Keep workflow/fragment metadata if needed by active code.
3. **Delete dead YAML files** — Remove `library/workflows/*.yaml`, `library/fragments/*.yaml` and their `__init__.py` files.
4. **Delete entire `maverick.dsl` package** — `rm -rf src/maverick/dsl/`
5. **Remove `lark` from dependencies** — Edit `pyproject.toml`.
6. **Run `uv sync`** — Verify clean dependency resolution without lark.
7. **Run `make lint` and `make typecheck`** — must pass. Test failures from dead test imports are expected and addressed in Phase 3.

### Phase 3: Clean Up Tests (P3 — User Story 4)

**Goal**: Move live tests, delete dead tests, verify suite passes.

**Sequence**:

1. **Move live executor tests** — `tests/unit/dsl/executor/` → `tests/unit/executor/`
2. **Move live checkpoint tests** — `tests/unit/dsl/checkpoint/` → `tests/unit/checkpoint/`
3. **Move live module tests** — `tests/unit/dsl/test_events.py` → `tests/unit/test_events.py`, same for `test_results.py`, `test_types.py`
4. **Update all test imports** — `maverick.dsl.*` → new locations
5. **Handle `test_errors.py`** — Update to test errors in their new `maverick.exceptions.workflow` location
6. **Delete remaining `tests/unit/dsl/`** — Everything else is dead (serialization, expressions, visualization, discovery, steps, prerequisites, context, streaming)
7. **Delete `tests/integration/dsl/`** — Entirely dead
8. **Run full test suite** — `make check` must pass
9. **Verify no `maverick.dsl` references remain** — `grep -r "maverick.dsl" src/ tests/` returns empty

### Phase 4: Documentation & CLAUDE.md Update

1. **Update CLAUDE.md** — Remove YAML DSL sections, update architecture diagram, update file organization
2. **Update constitution.md** — Remove guardrails #11 (workspace cwd threading for DSL) and #12 (DSL expression type safety) since DSL is gone. Update Appendix A split patterns.
3. **Verify `make ci` passes** — Final green check

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Transitive import through dead module | Phase 1 step 10: full test suite catches any missed dependency |
| Checkpoint files reference old module paths | Checkpoint data uses simple JSON (dicts/strings), not pickled classes — no impact |
| Test that imports from both live and dead DSL code | Careful test-by-test analysis during Phase 3 |

1. `make lint` — No broken imports
2. `make typecheck` — No type errors
3. `make test` — All active tests pass
4. `grep -r "maverick.dsl" src/ tests/` — Converges to zero by Phase 3 end
